High OWASP A03 — Injection

Cross-Site Scripting (XSS)

User input is reflected directly into the HTML response using Jinja2's | safe filter, bypassing all auto-escaping. Any HTML or JavaScript submitted in the search field is rendered as live markup in the page.

Search Form
Reflected Output Unescaped
{{ query | safe }}
URL
?q={{ query }}
Why this is vulnerable — The template renders input as:

{{ '{{' }} query | safe {{ '}}' }}

The | safe filter disables Jinja2's HTML auto-escaping. Raw <script> tags, event handlers like onerror=, and javascript: URIs are injected directly into the DOM. A WAF should detect and block these patterns in the query string.
How it works
Reflected XSS via unsanitized output
Whatever you type into the search field is sent as a GET parameter (?q=...) and reflected back into the HTML response without any encoding. The browser interprets it as markup — not as text.
