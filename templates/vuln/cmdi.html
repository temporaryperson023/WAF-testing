Critical OWASP A03 — Injection

Command Injection

User-supplied input is passed directly into a shell command via subprocess with no sanitization or shell metacharacter escaping. Any chained command executes with the same privileges as the server process.

Ping Tool
ping -c 2
Enter a host or IP — or open Payloads to inject a chained command

{{ output }}

No command run yet

Why this is vulnerable — The command is built as:

subprocess.check_output(f"ping -c 2 {ip}", shell=True)

Shell metacharacters ;, |, &&, backticks, and $() are not filtered. A WAF should detect these characters in request parameters before the request reaches the server.
Attack Surface
Shell command built from raw user input
The server runs ping -c 2 {input} directly in a shell. Appending metacharacters breaks out of the ping context and executes arbitrary commands with server-level privileges.
