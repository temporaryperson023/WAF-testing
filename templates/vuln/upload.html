High OWASP A05 — Security Misconfiguration

Unrestricted File Upload

Files are accepted without any content-type, MIME, or extension validation and saved directly to the server filesystem. Upload any file type including web shells, executables, and scripts.

Why this is vulnerable — The server calls: file.save(f"uploads/{filename}") with no checks on extension, MIME type, or filename sanitization. A WAF should block uploads of executable file types and malicious content-type headers.
Attack Vectors
What can be uploaded?
Without validation, an attacker can upload web shells, executables, scripts, or files with path-traversal filenames to write anywhere on the filesystem.
