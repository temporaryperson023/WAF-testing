Medium OWASP A01 — Broken Access Control

Broken Access Control

The admin panel at /vuln/admin/panel has no authentication, session validation, or role check. Any anonymous user who knows the URL gets full access to all user records and credentials. The WAF should detect and block direct access to this sensitive route.

Attack Simulation
🔓
Access the Unprotected Admin Panel
Clicking the button below sends a direct request to /vuln/admin/panel with no credentials or session token. A properly configured WAF should intercept this request and return a 403.
GET /vuln/admin/panel — no auth required
→ Access Admin Panel
If your WAF is active, this request will be blocked with HTTP 403.
If it loads — your WAF rule is missing or misconfigured.
Why this is vulnerable — The route is defined as:

@vuln_bp.route('/admin/panel')
def admin_panel():
    # no session check, no auth, no role validation
    return render_template("vuln/admin_panel.html", ...)

A WAF should block any unauthenticated request to /vuln/admin/* paths using path-based access rules or IP allowlisting.
Attack Surface
Sensitive route with no access enforcement
Broken Access Control occurs when an application exposes privileged functionality without verifying the requester's identity or role. The path /vuln/admin/panel returns full database contents to any visitor — no login, no token, no check of any kind.
01
Anonymous user requests the URL
GET /vuln/admin/panel — no session cookie, no token
02
WAF intercepts the request
Path-based rule matches /admin/* — returns 403 Forbidden
03
Without WAF — Flask serves the panel
No auth check in the route — full data exposed immediately
04
All sensitive data returned
Usernames, plaintext passwords, roles, and session tokens
Recommended WAF Rule ModSecurity
# Block all unauthenticated access to /admin/* paths SecRule REQUEST_URI "@beginsWith /admin/" \ "id:8001,phase:1,block,\ msg:'Unauthorized admin path access',\ severity:'CRITICAL'"
