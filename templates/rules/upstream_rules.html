Rule Type — Upstream Rule

Upstream Rules

Send a request to the main upstream and verify the WAF reroutes it to an alternate upstream server. The upstream field in the response reveals which server actually handled the request.

Trigger Request
GET /rules/upstream/test

Without a rule the response shows upstream: main-server. With the Upstream Rule active it should show upstream: alternate-server.

Response 
No request sent yet
Upstream vs Forwarder — A Forwarder Rule redirects at the path level within the same app. An Upstream Rule operates at the backend server level — routing traffic to a completely different upstream host or pool. Check the upstream field to confirm which server responded.
