Rule Type — Capture Rule

Capture Rules

Send a request with a user parameter and verify the WAF captures or logs the data before passing the request through to the backend. The response itself passes normally — capture is a logging-only action.

Send Request
GET /rules/capture/test?user=

The backend echoes back the user param and User-Agent header. The WAF should capture these values in its logs without modifying the response.

Backend Response 
No request sent yet
How to verify — Unlike other rules, Capture Rules don't change the response. Verification is done by checking your WAF's log dashboard — the captured user and User-Agent values should appear there after each request.
