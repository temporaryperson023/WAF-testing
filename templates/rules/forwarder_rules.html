Rule Type — Forwarder Rule

Forwarder Rules

Send a request to the primary backend and verify the WAF silently forwards it to a secondary backend instead. The backend field in the response reveals which server actually handled the request.

Trigger Request
GET /rules/forwarder/test

Without a rule the response shows backend: primary. With the Forwarder Rule active it should show backend: secondary.

Response 
No request sent yet
How to verify — Check the backend field in the JSON response. "primary" means the WAF rule is not active. "secondary" confirms the Forwarder Rule forwarded the request to /rules/forwarder/alt successfully.
WAF Config Guide Forwarder Rule
Rule Type Forwarder Rule
Match Path /rules/forwarder/test
Forward To /rules/forwarder/alt
Expected Behavior
Without WAF backend: primary
With WAF backend: secondary
Steps
Match on path: /rules/forwarder/test
Set forward destination to /rules/forwarder/alt
Send request — check backend field in response
01
Request sent to /rules/forwarder/test
Normally handled by the primary backend
02
WAF intercepts and forwards
Forwarder Rule silently routes to /rules/forwarder/alt
03
Secondary backend responds
Response contains backend: secondary

GET /rules/forwarder/test
