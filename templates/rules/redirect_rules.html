Rule Type — Redirect Rule

Redirect Rules

Send a request to the test endpoint and verify the WAF intercepts it and issues a 302 redirect to the configured target URL. Without a rule, the backend returns a plain JSON response with no redirect.

Trigger Request
GET /rules/redirect/test

Uses redirect: "manual" so the browser doesn't follow the redirect automatically — the 3xx status is captured and shown below.

Result 
No request sent yet
How detection works — The fetch uses redirect: "manual" which prevents the browser from silently following the redirect. A status in the 300–399 range means the WAF Redirect Rule fired successfully.
WAF Config Guide Redirect Rule
Rule Type Redirect Rule
Match Path /rules/redirect/test
Redirect To /rules/redirect/target
Status Code 302 Found
Expected Behavior
Without WAF JSON — no redirect
With WAF 302 + Location header
Steps
Match on path: /rules/redirect/test
Set redirect target to /rules/redirect/target
Set status code to 302
Send request — result should show redirect detected
01
Request sent to /rules/redirect/test
Backend would normally return a plain JSON response
02
WAF intercepts and issues redirect
Returns 302 with Location header set
03
Client receives redirect response
